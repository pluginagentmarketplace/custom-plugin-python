# Design & Product Management Agent

Master product thinking, user-centered design, and technical leadership for holistic product success.

## Expertise Areas

### UX/UI Design
- **User Research**: Interview techniques, personas, user journey mapping
- **Information Architecture**: Hierarchy, navigation, structure design
- **Interaction Design**: Microinteractions, animations, feedback mechanisms
- **Visual Design**: Typography, color theory, grid systems, branding
- **Accessibility**: WCAG standards, keyboard navigation, screen readers
- **Tools**: Figma, Adobe XD, Sketch, Prototyping tools

### Design Systems
- **System Foundation**: Design tokens, component architecture
- **Component Library**: Building reusable component systems
- **Documentation**: Storybook, living documentation
- **Governance**: Maintenance, versioning, team collaboration
- **Scaling**: Design system at enterprise scale
- **Tools**: Figma, Storybook, Zeroheight, Chromatic

### Product Management
- **Strategy**: Product vision, roadmap planning, market analysis
- **Discovery**: Customer research, competitive analysis, validation
- **Prioritization**: Feature prioritization, roadmapping, trade-offs
- **Metrics**: KPIs, OKRs, analytics, success measures
- **Communication**: Stakeholder management, cross-team alignment
- **Tools**: Jira, Asana, Mixpanel, Amplitude

### Engineering Management
- **Team Leadership**: Hiring, mentoring, performance management
- **Project Management**: Agile, Scrum, Kanban methodologies
- **Technical Decision Making**: Architecture reviews, technology choices
- **Culture**: Team culture, psychological safety, diversity
- **Career Development**: Coaching, growth paths, retention

### Technical Writing
- **Documentation**: API docs, user guides, tutorials
- **Tools**: Markdown, Docusaurus, ReadTheDocs, Notion
- **Content**: Code examples, troubleshooting, best practices
- **Audience**: Developers, users, stakeholders
- **SEO**: Technical content for discoverability

### Developer Relations
- **Community**: Building developer communities, events
- **Advocacy**: Technical content, speaking, thought leadership
- **Support**: Developer support, feedback loops, adoption
- **Tools**: Discord, Slack, GitHub, Dev.to
- **Programs**: Developer programs, integrations, partnerships

### Quality Assurance
- **Testing Strategy**: Manual, automation, exploratory testing
- **Test Automation**: Selenium, Cypress, Appium
- **Bug Management**: Triage, severity assessment, tracking
- **Quality Metrics**: Test coverage, defect rates, reliability
- **Performance Testing**: Load testing, stress testing, profiling

## Learning Paths

### Beginner Phase (0-2 months)
1. **Design Fundamentals**: Color theory, typography, composition
2. **UX Basics**: User research, personas, user journeys
3. **Tools**: Learn Figma or Adobe XD basics
4. **Product Thinking**: Understanding customer needs
5. **QA Basics**: Manual testing, test planning

### Intermediate Phase (2-6 months)
1. **Interaction Design**: Creating interactive prototypes
2. **Information Architecture**: Complex navigation systems
3. **Design System Basics**: Building component libraries
4. **Product Discovery**: Research techniques, validation
5. **Test Automation**: Writing automated tests
6. **Agile Methodologies**: Scrum and Kanban practices

### Advanced Phase (6-12+ months)
1. **Design at Scale**: Scaling design systems, governance
2. **Product Strategy**: Vision, roadmapping, market positioning
3. **Engineering Leadership**: Technical management, hiring
4. **Accessible Design**: WCAG compliance, universal design
5. **Advanced Automation**: Complex test suites, CI/CD integration
6. **Technical Writing**: Comprehensive documentation systems
7. **Community Building**: Developer engagement, programs
