# /project-builder

Explore hands-on projects and build your professional portfolio.

## Usage

- Learn through practical application
- Build real-world experience

- Practice with different technologies

## How to Choose a Project

**By Experience Level:**
- 0-1 year: Start with Beginner projects
- 1-3 years: Mix Intermediate with some Advanced
- 3+ years: Advanced projects and mentoring others

**By Interest:**
- Love design? Pick design/product projects
- Love building? Pick full-stack projects
- Love algorithms? Pick competitive programming

**By Time Available:**
- <5 hours/week: Choose 1-2 week projects
- 5-10 hours/week: Choose 3-4 week projects
- 10+ hours/week: Choose larger projects

**By Career Goal:**
- Freelance: Build diverse portfolio
- Startup: Full-stack or specialized depth
- Enterprise: Architecture and scale projects

## Project Template Structure

```
project-name/
├── README.md (description, setup, demo)
├── docs/
│   ├── REQUIREMENTS.md
│   ├── ARCHITECTURE.md
│   └── API_DOCS.md
├── src/
│   ├── components/
│   ├── services/
│   └── utils/
├── tests/
│   ├── unit/
│   └── integration/
├── .github/
│   └── workflows/ (CI/CD)
├── docker/
│   └── Dockerfile
└── Makefile (common commands)
```

## Portfolio Tips

**For Each Project Include:**
1. **Live Demo** - Working application
2. **GitHub Repo** - Clean code with good commits
3. **Case Study** - Problem, solution, learnings
4. **Technical Details** - Tech stack, architecture decisions
5. **Results** - Metrics, performance, user feedback

**Showcase on:**
- Personal portfolio website
- GitHub profile
- LinkedIn
- Dev.to articles
- Product Hunt (if it's a product)
